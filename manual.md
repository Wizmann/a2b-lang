# A=B 语言简明规范

A=B 是对单行 ASCII 字符串进行改写的语言。程序是按书写顺序排列的规则列表。

本文不讨论 `#...` 和 `/*...*/` 注释。

## 1. 程序语法

每个非空行必须恰好包含一个 `=`，形式为：

```text
左侧=右侧
```

简化文法：

```text
左侧 ::= [ "(start)" | "(end)" | "(once)" ] 文本
右侧 ::= [ "(start)" | "(end)" | "(return)" ] 文本
文本 ::= 零个或多个可用 ASCII 字符
```

规则正文只能使用 ASCII。`=`、`#`、`(`、`)` 是保留字符，不能属于“文本”。没有转义语法。空格、`^`、`$` 等其他 ASCII 字符都是普通字符，不应自动删去。

“文本”可以为空，因此 `a=`、`=a`、`=(return)ok` 都是合法规则。每侧至多有一个关键字，关键字只能出现在该侧开头。

## 2. 执行过程

令当前字符串为输入字符串。反复执行：

1. 从第一条规则开始向下查找第一条可匹配的规则。
2. 执行该规则一次；普通匹配只处理最靠左的一处，不会一次替换全部。
3. 若没有触发 `(return)`，从第一条规则重新查找。
4. 若所有规则都不可匹配，结束并输出当前字符串。

因此，规则顺序会影响结果，且每次改写后都必须从第一条规则重新扫描。

## 3. 左侧关键字：如何匹配

设左侧文本为 `A`：

- `A=B`：`A` 可出现在任意位置；选择最靠左的一处。
- `(start)A=B`：仅当当前字符串以 `A` 开头时匹配。
- `(end)A=B`：仅当当前字符串以 `A` 结尾时匹配。
- `(once)A=B`：匹配方式与普通 `A` 相同，但该规则成功执行一次后，本次程序运行中永远跳过它。未匹配时不会消耗这次机会。每次用新输入重新运行程序时，状态重置。

空文本始终可匹配：普通空文本和 `(start)` 空文本位于字符串开头，`(end)` 空文本位于字符串结尾。

## 4. 右侧关键字：如何改写

设匹配到的左侧片段为 `A`，右侧文本为 `B`：

- `A=B`：在原位置用 `B` 替换 `A`。
- `A=(start)B`：删除匹配到的 `A`，再把 `B` 添加到整个字符串开头。
- `A=(end)B`：删除匹配到的 `A`，再把 `B` 添加到整个字符串结尾。
- `A=(return)B`：立刻终止程序并直接输出 `B`。此输出与当前字符串的其余内容无关。

左右关键字可以组合。例如：

```text
(start)a=(end)X
```

它仅匹配开头的 `a`，删除该 `a`，再把 `X` 添加到字符串末尾。

## 5. 最小示例

程序：

```text
a=A
b=B
c=C
```

输入 `abac`，依次得到 `Abac`、`AbAc`、`ABAc`、`ABAC`，最终输出 `ABAC`。

无规则的空程序直接输出原输入。
